01 / Principle

The files are the project.

Markdown on disk is the source of truth. AI sessions are interchangeable compute that read the same brain. Close any chat at any time — the project is intact.

02 / Principle

A duck can't invent your business logic.

If the rule isn't written down, the AI invents one. Specifications come from the human; execution comes from the model. Improvisation bias dies at the dry-run gate.

03 / Principle

Cleaner context beats more context.

Big context windows don't fix vibe coding — they extend it. Each session gets only the files it needs for the job in front of it. Nothing more.

§07 — Applied / Dual-mode orchestration

Architect plans. Builder builds.

Two modes, any model. The framework doesn't care which AI you use — it cares which mode you've invoked.
Mode 01 / Planning

The Architect.

Senior tech lead who writes the plan but never touches the keyboard.
  • Asks clarifying questions before doing anything
  • Produces requirements.md, blueprint.md, acceptance.md
  • Reviews dry runs, audits builder output
  • Never writes application source code
  • Never reads live runtime logs
Mode 02 / Execution

The Builder.

Engineer who writes exactly what the blueprint says — and stops before changing files.
  • Runs a read-only dry run first, then waits
  • Implements only files listed in the blueprint
  • Logs every change in implementation_log.md
  • Never invents business logic outside DOMAIN.md
  • Never skips the dry-run gate
