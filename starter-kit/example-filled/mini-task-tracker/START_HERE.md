# Start Here - Mini Task Tracker

Starter kit version: 0.1.0.

Use this file as the first-session checklist for Mini Task Tracker. The example already shows completed ADDF files for a small local-only task app.

1. Copy this starter kit into the root of Mini Task Tracker.
2. Fill `SECURITY.md` - list the files that must never be loaded.
3. Fill `AGENTS.md` for the Mini Task Tracker.
4. Draft `DOMAIN.md` - confirm local-only scope and task terminology.
5. Initialize `STATE.md` - set scale and mode.
6. Run Research Mode if unknowns exist - localStorage limits were researched for v0.1.
7. Run Design Mode to complete the project brain.
8. Run Design Mode to generate the Sprint 001 sprint pack.
9. Run Develop Mode at Permission Level 0 - dry run only.
10. Review `dry_run.md` - check every file and command listed.
11. Authorize Develop Mode - use the exact authorization message: "Dry run approved. / Permission Level [LEVEL] authorized. / Proceed according to requirements.md, blueprint.md, acceptance.md, and dry_run.md."
12. Verify output against acceptance criteria.
13. Update `STATE.md`.
14. Commit.
15. Close or checkpoint the AI session.

## Next Sessions

The model has no memory between sessions. The files are the project memory. Load the project brain files at the start of every Mini Task Tracker session.
