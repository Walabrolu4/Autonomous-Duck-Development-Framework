# STYLE_GUIDE.md - Mini Task Tracker

## 1. Voice

1. Use direct, plain language.
2. Describe behavior in terms of operator actions.
3. Avoid product-management jargon.

## 2. Banned Words

- just
- simply
- obviously
- easy
- smart

## 3. Canonical Terminology

| Term | Use |
|---|---|
| Task | Use instead of item or todo. |
| TaskList | Use instead of list when referring to stored task data. |
| Complete | Use instead of done or finished. |

## 4. File Naming

- Use lowercase file names.
- Use hyphens for docs if needed.
- Do not use spaces in file or folder names.

## 5. Document Structure

1. Put current state before future ideas.
2. Separate v0.1 facts from v0.2 questions.
3. Link decisions instead of repeating long rationale.

## 6. No Emoji Rule

Do not use emoji in project files, commit messages, or AI prompts.

## 7. Full Brand Guide

No separate brand guide exists for Mini Task Tracker v0.1.
