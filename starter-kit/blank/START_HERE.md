# Start Here - `[YOUR PROJECT NAME]`

Starter kit version: 0.1.0.

Use this file as the first-session checklist for adopting ADDF in `[YOUR PROJECT NAME]`. Work through the steps in order and keep the files as the project memory between AI sessions.

1. Copy this starter kit into the root of `[YOUR PROJECT NAME]`.
2. Fill `SECURITY.md` - list the files that must never be loaded into AI tools.
3. Fill `AGENTS.md` - confirm the three modes and add your safe-to-load file list.
4. Draft `DOMAIN.md` - write your project identity, scope, operators, terms, and rules.
5. Initialize `STATE.md` - set the active scale, current mode, current status, and next step.
6. Run Research Mode if unknowns exist - skip this only if the project direction is already clear.
7. Run Design Mode to complete the project brain.
8. Run Design Mode to generate the Sprint 001 sprint pack.
9. Run Develop Mode at Permission Level 0 - dry run only.
10. Review `dry_run.md` - check every file, command, dependency, risk, and ambiguity listed.
11. Authorize Develop Mode - use the exact authorization message: "Dry run approved. / Permission Level [LEVEL] authorized. / Proceed according to requirements.md, blueprint.md, acceptance.md, and dry_run.md."
12. Verify output against the acceptance criteria in `acceptance.md`.
13. Update `STATE.md` with the verified status and next step.
14. Commit the reviewed project files.
15. Close or checkpoint the AI session.

## Next Sessions

The model has no memory between sessions. The files are the project memory. At the start of every session, load `AGENTS.md`, `STATE.md`, `DOMAIN.md`, and any relevant sprint pack or project brain files before asking the model to act.
