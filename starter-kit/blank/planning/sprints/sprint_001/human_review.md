## How to use this file

This file is generated by the AI at the end of a sprint's Develop Mode. The AI reads `acceptance.md` for this sprint and produces one section per acceptance group — matching the section numbers and titles exactly. The AI does not fill in verdicts or the approval signature. The human reviewer opens `acceptance.md`, works through each check directly, then returns here to record verdicts and sign off.

---

## How this file is generated

To generate this file for any sprint, enter Design Mode and run:

> "You are in Design Mode. Read `planning/sprints/sprint_[NUMBER]/acceptance.md`. Produce `human_review.md` for this sprint with one section per acceptance group, matching the section numbers and titles from `acceptance.md`. Each section must contain a Pass/Fail/Partial verdict field and an issues field. Do not fill in verdicts, issues, or the approval signature. Stop when the file is written."

Replace `[NUMBER]` with the active sprint number before running.
