# ADDF Starter Kit

## What this is

The ADDF Starter Kit is the primary adoption artifact for the Autonomous Duck Deployment Framework. Copy one of these kit variants into a project to give AI sessions a file-backed operating contract, project memory, sprint structure, and review gates before implementation begins.

## Two variants

| Variant | Use when |
|---|---|
| `blank/` | You are starting a new project and want clean templates with placeholders. |
| `example-filled/mini-task-tracker/` | You want to see a small completed ADDF project brain before filling in your own files. |

## How to use the blank kit

1. Copy the contents of `starter-kit/blank/` into the root of your project.
2. Open `START_HERE.md` and confirm the starter kit version is `0.1.0`.
3. Fill `SECURITY.md`, `DOMAIN.md`, and `STATE.md` before sharing files with an AI tool.
4. Use Design Mode to complete the project brain and write the first sprint pack.
5. Use Develop Mode only after a dry run has been reviewed and authorized.

## Minimum viable ADDF

For the smallest useful setup, keep these files:

- `AGENTS.md`
- `DOMAIN.md`
- `STATE.md`
- `COMMANDS.md`
- `SECURITY.md`
- `planning/sprints/sprint_001/`

## Full ADDF additions

For a complete project brain, add:

- `DECISIONS.md`
- `QUESTIONS.md`
- `RISKS.md`
- `GIT_STRATEGY.md`
- `PROMPT_CHANGELOG.md`
- `STYLE_GUIDE.md`
- `docs/`
- `research/`
- `planning/backlog.md`
- `planning/releases/`
- `planning/features/`
