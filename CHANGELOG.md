## Changelog conventions

Entries are grouped by release. Within a release, changes are grouped by type.

| Section | Contents |
|---|---|
| **Added** | New files, new sections, new prompts, new examples. |
| **Changed** | Changes to existing content that are not bug fixes. |
| **Fixed** | Corrections to errors in documentation, templates, or prompts. |
| **Deprecated** | Content that will be removed in a future release. |
| **Removed** | Content that was removed. |
| **Security** | Changes related to security, secrets, or safe-to-load rules. |

Breaking changes — prompt updates that change authorization behavior, starter kit structure changes that require operator migration, terminology changes — must be noted as `**Breaking:**` at the start of the line.
